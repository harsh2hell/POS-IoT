##  Introduction

This project is a **web-based POS system** aimed at small stores, kiosks, and learning purposes.  
It focuses on **clarity, consistency, and extendability**, ensuring that product UI remains fixed while all business logic and data are handled dynamically from the backend.

The system is suitable for:
- Retail POS experiments
- SuperMart / Retail style Billing
- Academic & portfolio projects
- Flask backend practice

---

##  Features

###  Customer Experience
- Fixed product card layout (UI consistency)
- Dynamic product listing from database
- Add to cart / remove from cart
- Auto price calculation
- Mobile-friendly layout

###  Admin / Backend
- Product CRUD operations
- Stock availability control
- Category-based product grouping
- Clean Flask routing
- Jinja2 template rendering

###  System Design
- Flask MVC-style structure
- SQLite database (easy to upgrade)
- Stateless routes (ready for API mode)
- Future-ready for mobile apps
